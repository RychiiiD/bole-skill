Collect enterprise qualification lists from government websites using MCP browser tools.

## Input

Read `data/enterprise/.enterprise_sources.json` (generated by `bole-enterprise-sources`) for the list of qualification sources to collect:

```json
{
  "city": "{城市名}",
  "qualification_sources": [
    {
      "qualification_type": "专精特新",
      "source_name": "吉林省工信厅_2025省级专精特新",
      "url": "http://...",
      "page_type": "pdf_download"
    }
  ]
}
```

- `qualification_sources` — confirmed sources with URL, type, and source_name
- `page_type` determines which extraction approach (Case A/B/C below)
- Iterate through all sources in the list, one by one

## ★ 强制自检协议（必须遵守，违反即违规）

### ★ 采集前合规三步检查（必须执行，不可跳过）

开始采集前依次执行以下检查：

#### 1. robots.txt 合规检测

```yaml
首次导航到该来源前，先检测来源站点的 robots.txt:
  python scripts/pipeline/pipeline.py robots-check <source_url>
  ├─ exit(0) → robots.txt 允许或不存在，继续
  └─ exit(1) → Disallow 匹配，暂停并向用户展示检测报告
                └─ 用户确认后方可继续，用户不同意则跳过该来源

同一来源下不同页面不重复检测，仅首次导航时执行一次。
```

#### 2. 速率限制约束

每次导航/翻页前先运行 rate-tick：

```yaml
python scripts/pipeline/pipeline.py rate-tick <source_id>
├─ exit(0) → 速率门禁通过，可以发送请求
└─ exit(1) → 速率限制触发，等待后重试，不得自行绕过

约束:
  - 连续 5 页后停顿 ≥ 3 秒
  - 单来源不超过 180 页/小时
```

#### 3. 下载链接检测（代码级，必须执行，不可跳过）

Before collecting from any enterprise/government source URL, run the **code-level download-link detector** (`robot_scan.py`) to check whether the source page provides visible download links to data files (.xlsx/.xls/.pdf/.doc/.docx). A government website that publishes data via a visible download link is actively providing that data. **Only sources with downloadable data files are collected — no AI judgment, no terms-scraping, no pattern matching.**

##### 执行步骤

```yaml
1. 运行 robot_scan.py 执行下载链接检测（每次新建一个来源时执行一次）:
   python scripts/enterprise/robot_scan.py \
     --url "{source_url}" \
     --output "data/enterprise/raw/.robot_check_{safe_source_name}.json" \
     --source-name "{source_name}"

   safe_source_name: 将 source_name 中的 / \ . 替换为下划线

2. 运行代码门验证检测结果:
   python scripts/pipeline/pipeline.py robot-check enterprise "{source_name}"
   ├─ exit(0) → 有下载链接 (collectable=true) → 继续采集
   └─ exit(1) → 无下载链接 → 跳过该来源
                 → 输出中会展示下载链接数和原因
                 → 在来源间放行协议中展示为 ⛔ 跳过（无下载链接）

3. ⚠️ 如 robot_scan.py exit(1)（HTTP 错误/无法访问）:
   - 检查 URL 是否正确
   - 如 URL 有效但网络不可达，该来源本身即无法采集，按正常跳过逻辑处理
   - 不得跳过检测直接采集
```

#### 检测原理

`robot_scan.py` 通过 `requests` 获取页面 HTML，用正则扫描 `<a href>` 链接指向的可下载数据文件。**不含 BS4、不含 config.yaml 模式匹配、不含 AI 判断。**

```
扫描方式: requests + regex 匹配 <a href="...xlsx"> 等下载链接
匹配范围: .xlsx / .xls / .pdf / .doc / .docx
阻断条件: 页面无任何可下载数据文件链接 → collectable=false
```

#### 特殊说明

- 此检测在**首次导航到该来源时**执行一次，翻页/重试不需要重复执行
- 如 `.robot_check_{safe_name}.json` 已存在且 `collectable=true`，跳过此步骤直接进入采集
- `collectable=false` 的来源在「来源间人工放行协议」中展示为"⛔ 已跳过（无下载链接）"
- ★ `pipeline.py verify enterprise collect` 会校验每条数据文件是否有对应 `.robot_check_*.json` 且 `collectable=true`，缺失则 exit(2) STOP

### 导航后自检 checklist

每次 `browser_navigate` 后，**必须**执行以下自检流程，不得跳过、不得臆断：

```yaml
1. browser_snapshot          → 确认页面渲染结果
2. 检查页面是否包含以下禁止状态:
   - "登录"/"注册"/"验证码"/"扫码"  → 暂停，提示用户手动操作
   - "不存在"/"没有找到"/"暂无"     → 此为客观结果，如实记录，不臆断原因
   - 页面跳转到 about:blank        → 页面异常，尝试重新导航
3. 检查页面类型是否与预期一致:
   - 预期网页表格 → 页面是否有 table/列表结构？
   - 预期下载链接 → 页面是否有 .pdf/.xls/.xlsx 下载按钮？
   - 预期查询页   → 是否有下拉框/查询按钮？
4. 检查结果分类处理:
   - 正常显示企业名单/表格 → 继续采集
   - 登录/验证码拦截       → 暂停脚本，提示用户手动处理
   - 空结果（暂无数据）    → 记录到日志，跳过该来源
   - 页面异常/跳转        → 尝试重新导航一次，仍失败则跳过
```

### 下载/提取后自检 checklist

每次执行完下载或表格提取后，**必须**执行：

```yaml
1. 文件验证:
   - 文件是否已保存到 data/enterprise/raw/qualification_lists/?
   - 文件大小是否 > 0 字节？
   - 如果是 PDF/Excel：是否能用 Python 正常打开解析？
2. 数据完整性:
   - 提取的企业名称列表是否非空？
   - 表格类页面：是否提取了所有行（包括所有分页）？
   - 下载类文件：解析出的行数是否与文件预期一致？
3. 记录:
   - 记录提取的企业数量
   - 如企业数为 0 → 确认是"名单为空"还是"提取失败"，不得混淆
```

### 禁止行为清单

| ❌ 禁止 | ✅ 正确做法 |
|---------|------------|
| 看到空结果就臆断"该来源暂无企业名单" | 先确认是否需要登录/切换年份才能显示 |
| 只提取第一页就结束 | 必须全量翻页，提取所有行 |
| 下载 PDF 后不解析直接跳过 | 必须解析文件提取企业名称 |
| 以"可能""也许""大概"推测页面状态 | 用 browser_snapshot 获取客观事实 |
| 跳过未搜索到的资质类型 | 如实记录"未找到该类型公示" |
| 遇到偏离原始流程的决定（跳过来源/资质类型/步骤、缩减采集量、用部分结果替代完整结果、流程未明确规定时的替代方案） | 必须暂停并询问用户，不得自行决定 |
| **PNG/JPEG 截图代替数据文件保存** | PNG/JPEG 不是有效产出格式，浏览器截图不能替代可解析的数据文件 |
| **PDF 解析失败后走 OCR 弯路（渲染 PDF 每页为 PNG 再识别）** | 运行 `scripts/enterprise/pdf_extract.py` 自动四层文本提取 → 仍失败则 Playwright 提取 text-layer → 仍无效才 `parse_failed` |
| **先下载 PDF 不检查页面有无其他格式** | 下载前先用 browser_snapshot 观察页面所有可下载文件，优先选可解析格式 |

## ★ 文件格式规则（硬性规定，不得违反）

### 允许的产出格式

| 格式 | 说明 | 来源 |
|:-----|:-----|:-----|
| `.csv` | 解析后输出的结构化数据 | 自行生成 |
| `.json` | 结构化数据，极少使用 | 自行生成 |
| `.xls` / `.xlsx` | 从页面下载的原始 Excel | 浏览器点击下载 |
| `.pdf` | 从页面下载的原始 PDF | 浏览器点击下载 |
| `.doc` / `.docx` | 从页面下载的原始 Word | 浏览器点击下载 |
| `.txt` | 附注/说明文件 | 自行生成 |

### 禁止的格式

`.png`、`.jpg`、`.jpeg`、`.gif`、`.bmp` 等图片格式**不得出现在 `qualification_lists/` 目录下**。pipeline.py verify 会扫描此目录，发现图片文件则 exit(2) STOP — 图片无法被解析为企业名录。

### 格式优先级

下载前必须评估页面提供的所有可下载文件，按以下优先级选择：

```
xlsx / xls（优先，结构化数据最完整）
  → docx / doc（次选，可解析）
  → PDF（三选，解析有风险）
  → 网页表格（兜底，直接提取）
```

同一页面有多个格式时，**不得先下载低优先级的格式（如 PDF）而不查看是否有高优先级格式（如 xlsx）**。

## ★ 下载行为定义（硬性规定）

"下载"在伯乐企业管线中专指：**模拟人工点击页面上可见的下载链接/按钮，由浏览器驱动下载文件到本地**。

以下行为**不属于"下载"**，禁止作为数据获取手段：
- ❌ 将浏览器当前页面另存为 PNG/截图
- ❌ 复制页面文本拼成数据文件（内容不变可以，但格式不是截图）
- ❌ 用 PDF 每页渲染为图片再 OCR

## Process (真实采集模式)

For each discovered government URL, use the available browser tools (MCP Playwright) to access the page and extract the enterprise list.

**每个来源采集前，必须先执行「采集前合规三步检查」（robots.txt → rate-tick → 下载链接检测）。**
**每个来源采集完成后，执行「来源间人工放行协议」，等待用户确认后再开始下一来源。**

### 采集前合规三步检查

对每个来源，在开始浏览器操作**之前**，依次执行：

```yaml
1. robots.txt 合规检测（首次导航到该来源时执行一次）:
   python scripts/pipeline/pipeline.py robots-check <source_url>
   ├─ exit(0) → 允许或不存在，继续
   └─ exit(1) → Disallow 匹配，暂停等待用户决策

2. 速率限制（每次导航/翻页前执行）:
   python scripts/pipeline/pipeline.py rate-tick <source_id>
   ├─ exit(0) → 通过，继续
   └─ exit(1) → 限制触发，等待后重试

3. 下载链接检测（首次导航到该来源时执行一次）:
   python scripts/enterprise/robot_scan.py \
     --url "{source_url}" \
     --output "data/enterprise/raw/.robot_check_{safe_name}.json" \
     --source-name "{source_name}"
   ├─ exit(0) → 检测完成，检查输出中的 collectable 字段
   └─ exit(1) → HTTP 错误，该来源不可达，跳过

   运行代码门:
   python scripts/pipeline/pipeline.py robot-check enterprise "{source_name}"
   ├─ exit(0) → 有下载链接 (collectable=true)，继续采集
   └─ exit(1) → 无下载链接，跳过该来源
```

### Case A — 页面有下载链接（PDF/Excel/Word）

1. 用 `browser_navigate` 打开页面
2. **列出页面上所有可下载文件**（用 `browser_snapshot` 观察下载链接/按钮）
   - 记录每个文件的链接文本、URL 后缀、文件格式
   - 示例输出：`找到 3 个下载: 名单.xlsx, 附件.pdf, 通知.doc`
3. **按格式优先级选择**：xlsx > docx > PDF > 网页表格
   - 有 xlsx 链接 → 下载 xlsx（优先，结构化最完整）
   - 只有 PDF → 下载 PDF
   - 同时有 xlsx + PDF → **必须**下载 xlsx，不得先下 PDF
4. 点击下载链接/按钮，文件由浏览器保存到 `data/enterprise/raw/qualification_lists/`
   - **点击下载是模拟人工操作**：找到页面上可见的 `<a href="...">下载</a>` 或下载按钮，`browser_click` 触发
   - 不得将页面另存为截图、不得逐页渲染 PDF 为图片
5. 解析下载的文件提取企业名称：
   - **Excel** (xlsx/xls)：`import openpyxl` 或 `import xlrd` 读取
   - **Word** (docx)：`import docx` 读取表格
   - **PDF**：运行 `python scripts/enterprise/pdf_extract.py` 自动四层递进提取
     ```bash
     python scripts/enterprise/pdf_extract.py <pdf_path> \
       --output data/enterprise/raw/qualification_lists/<filename>.csv \
       --source-name "来源_年份资质" \
       --category "专精特新"
     ```
     - **exit 0** → CSV 就绪，继续下一步
     - **exit 1** → **必须按以下顺序执行兜底，不得跳过 Playwright 直接 parse_failed**：
       1. 生成 Playwright 提取代码：
          ```bash
          python scripts/enterprise/pdf_extract.py --playwright-code <PDF_URL>
          ```
          脚本输出 `browser_evaluate` 用 JS 代码和操作步骤。
       2. 按指引执行：
          - `browser_navigate` 打开 PDF 的原始 URL
          - `browser_snapshot` 确认页面渲染
          - `browser_evaluate` 执行 JS 代码提取 text-layer 文本
       3. 检查返回的 JSON：
          - `"status": "ok"` → 提取成功，保存文本到临时文件，再运行：
            ```bash
            python scripts/enterprise/pdf_extract.py --import-text <tmp.txt> \
              --output data/enterprise/raw/qualification_lists/<filename>.csv \
              --source-name "来源_年份资质" \
              --category "专精特新"
            ```
          - `"status": "garbled"` 或 `"no_text_layer"` → Chrome 也无法解码
       4. 仍无效 → 回到页面检查是否有其他格式的下载链接
          - 有 → 下载备用格式重新解析
          - 无 → 如实记录 `parse_failed`，保存 `.txt` 附注说明原因
6. Write to CSV with format:
   ```csv
   company_name,enterprise_category,source_name
   长春XX科技有限公司,专精特新,吉林省工信厅_2025省级专精特新
   长春YY数据有限公司,专精特新,吉林省工信厅_2025省级专精特新
   ```

### Case B — 网页表格展示

1. Open the page using browser tools
2. Read the table content row by row
3. If paginated, click the next page button and continue
4. Write all rows to CSV (same format as above)
5. **全量原则**: 提取表格中所有企业，不按任何条件筛选

### Case C — 需要交互

Some government sites require:
- Selecting a year/region from dropdown
- Clicking "查询" button
- Solving a simple CAPTCHA

In these cases:
1. Use browser tools to interact with the page elements
2. Guide the user if manual interaction is needed (e.g., solving CAPTCHA)
3. Extract the results

## Output

Each government source gets its own CSV file:

```
data/enterprise/raw/qualification_lists/
├── 吉林省工信厅_2025省级专精特新.csv
├── 长春市科技局_高新技术企业.csv
├── 吉林省科技厅_瞪羚企业.csv
```

全部采集完成后标记：
```bash
python scripts/pipeline/pipeline.py complete enterprise collect
```

Each CSV has the same format:
```
company_name,enterprise_category,source_name
```

Report back:
- How many sources were collected
- Total enterprises extracted
- File paths for each
- Any parse failures (PDF that couldn't be extracted, with reason)
- No PNG/JPEG files were created in qualification_lists/
