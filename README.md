这和网上直接搜索有什么不同？

你在招聘网站上看到的，是平台想让你看到的。伯乐给你的，是它替你筛过的。
一边是几十个结果堆在面前真假自辨，一边是排序清楚标注理由。


换城市换岗位直接换说法：

## 能做什么

| 能力 | 说明 |
|------|------|
| **多平台采集** | BOSS/猎聘/前程无忧/智联/拉勾，多个关键词遍历到末页，空结果注明原因 |
| **企业资质核验** | 根据目标城市，从公示网站公开提供的下载附件获取企业资质名单，交叉比对岗位所属企业 |
| **透明评分** | 薪资/经验/学历/相关度四维评分 + 企业质量加分，权重可调且可持久化 |
| **对话追问** | 流程完成后像聊天一样追问：查公司背景、对比岗位、按条件筛选 |

## 工作方式

三条管线依次执行，每步受 `pipeline.py` 强制校验：

**1. 岗位采集管线** — 在 5 个主流招聘网站用关键词搜索，逐来源逐关键词翻页到末页。遇到登录或验证码暂停等待用户处理，空结果注明原因不遗漏。

**2. 企业资质管线** — 针对目标城市，从公开渠道采集企业资质名单（专精特新、高新技术企业等），清洗去重后按等级评分。有资质的企业自动加分；没有的不扣分。

**3. 富化匹配管线** — 将岗位数据与企业资质交叉匹配，为评分提供企业质量维度参考。

## 使用须知

伯乐是一个个人开源项目，供求职者收集和整理公开招聘信息。

- 所有采集操作在用户自己的浏览器中进行。遇到登录页面或验证码时管线会自动暂停，不会自动绕过。
- 采集频率受代码约束（连续翻页不超过 5 页后自动暂停），不以超过正常浏览极限的频率请求。
- 所有数据在用户本地处理，不上传至任何第三方服务器。岗位原始数据自采集日起超过 7 天自动清理。
- 本项目的唯一目的是为求职者个人提供岗位信息聚合参考，**不得用于商业目的或大规模数据采集**。

## 伯乐的边界

- 数据质量取决于来源平台，平台缺失的信息伯乐无法补全
- 采集依赖浏览器和网站结构，平台改版可能导致中断
- 企业资质核验仅采集公开下载附件（xlsx/pdf 等），无下载链接的来源跳过
- 评分是个人偏好参考，结果因人而异（权重可调）

## 「赛博求职」背后的故事

如今 Claude Code、Cursor、Codex——每个新工具都说要替你写代码。但我意识到一件很荒唐的事：既然 AI 快替我上班了，那为什么工作还得自己找？

越是信息时代，信息洼地越是比你想象得更深。

最开始只是为了解决一个问题：信息太散，真假难辨。后来改着改着就变成了伯乐.skill。在 AI 让一切变快的时代，伯乐选择慢一点，稳一点。

bole 是树干的意思——不起眼，枝叶却倚着它往上长。

伯乐，就是求职路上的树干。
